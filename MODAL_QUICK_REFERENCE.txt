CRITICAL DIFFERENCES (Only 3 main ones)
========================================

1. PAYMENT METHOD & STATUS DROPDOWNS (ORDER ONLY)
   Order modal has 2 extra dropdown fields that quotation does not have:
   - Payment Method (결제방법) - options: 현금, 계좌이체, 어음, 카드
   - Status (상태) - options: 작성중, 발주완료, 입고완료
   
   These appear between supplier selection and title field in the 2-column grid.
   Quotation modal doesn't have these fields.

2. TABLE BODY ID NAMING PATTERN
   Quotation: "quotationDetailTableBody" (simple pattern)
   Order:     "newOrderDetailTableBody" (includes "new" prefix)
   
   This is inconsistent with the pattern used for other IDs in order modal.

3. SUBMIT BUTTON TEXT
   Quotation: "견적서 등록"
   Order:     "저장"
   
   Consider standardizing to one pattern.

FORM SUBMISSION
===============
Quotation: <form id="quotationForm" onsubmit="submitQuotation(event)">
Order:     <form id="orderForm" onsubmit="submitNewOrder(event)">

Both use submit event handlers in their respective JS files.


TABLE STRUCTURE
===============
Both use identical 9-column table:
1. 순번 (row number)
2. 자재코드 (material code)
3. 자재명 (material name)
4. 규격 (specification)
5. 수량 (quantity)
6. 단가 (unit price)
7. 공급가 (supply price)
8. 부가세 (VAT)
9. 삭제 (delete button)

Footer shows totals for supply price, VAT, and grand total.
