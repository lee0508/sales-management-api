VISUAL DIFFERENCES
==================

Only 3 visual differences:

1. HEADER TITLE
   Quotation: "견적서 작성"
   Order:     "발주서 작성"

2. CUSTOMER/SUPPLIER SECTION
   Quotation: "매출처 *" + label "선택된 매출처:" 
   Order:     "매입처 *" + label "선택된 매입처:"

3. EXTRA ROW IN ORDER MODAL (★ marked above)
   Quotation: Does NOT have this row
   Order:     Has "결제방법" (Payment Method) and "상태" (Status) dropdowns
   
4. SECTION HEADERS
   Quotation: "견적 상세내역"
   Order:     "발주 상세내역"

5. SUBMIT BUTTON TEXT
   Quotation: "견적서 등록"
   Order:     "저장"

Everything else is IDENTICAL.


GRID COLUMN STRUCTURE
=====================

Quotation (5 rows):
  Row 1: [Date] [DeliveryDate]
  Row 2: [Customer Search] (full width)
  Row 3: [Title] (full width)
  Row 4: [Remarks] (full width)

Order (5 rows):
  Row 1: [Date] [DeliveryDate]
  Row 2: [Supplier Search] (full width)
  Row 3: [PaymentMethod] [Status]           ← EXTRA ROW IN ORDER
  Row 4: [Title] (full width)
  Row 5: [Remarks] (full width)

RESPONSIVE BEHAVIOR
===================

Max width: 1200px (both modals)
Height: 90vh (both modals)

When content overflows:
- Detail table: horizontal scroll (overflow-x: auto)
- Form area: vertical scroll (on parent div)
- Header: sticky (position: sticky, top: -32px, z-index: 10)


MODAL OVERLAY
=============

Both modals use:
- class="modal" with style="display: none;"
- Covers entire viewport (fixed positioning)
- Shows centered modal-content with backdrop

Modal trigger functions:
- Quotation: openQuotationModal()
- Order: openOrderModal()

Modal close functions:
- Quotation: closeQuotationModal()
- Order: closeOrderModal()
